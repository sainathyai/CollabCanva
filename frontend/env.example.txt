# Frontend Environment Variables

# WebSocket URL
# Development: ws://localhost:8080
# Production: wss://your-backend-domain.com
VITE_WS_URL=ws://localhost:8080

# Firebase Configuration
# Get these from Firebase Console > Project Settings > General
VITE_FIREBASE_API_KEY=your-api-key-here
VITE_FIREBASE_AUTH_DOMAIN=your-project-id.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your-project-id
VITE_FIREBASE_STORAGE_BUCKET=your-project-id.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your-sender-id
VITE_FIREBASE_APP_ID=your-app-id

# ====================
# DEPLOYMENT NOTES
# ====================

# For Vercel:
# 1. Add these environment variables in Vercel dashboard (Project Settings > Environment Variables)
# 2. Make sure VITE_WS_URL points to your backend WebSocket URL (use wss:// for secure connection)
# 3. All VITE_* variables are built into the frontend bundle at build time

# For Netlify:
# 1. Add these environment variables in Netlify dashboard (Site settings > Build & deploy > Environment)
# 2. Same rules as Vercel apply

# For Local Development:
# 1. Copy this file to .env in the frontend directory
# 2. Fill in your Firebase configuration
# 3. VITE_WS_URL defaults to ws://localhost:8080

# Note: OpenAI API key is now managed on the backend via AWS Secrets Manager
# No need to configure VITE_OPENAI_API_KEY in the frontend
